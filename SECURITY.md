# 安全策略

## 支持范围

`novel-assistant` 是本地安装的写作 skill 包。公开发布包不应包含个人写作资产、原始小说文本、访问凭据、内网地址或本地可选扩展。

安全问题包括但不限于：

- 公开发布清理或审计遗漏，导致个人资产、凭据或本地路径泄露；
- 工作流写锁、章节提交或路径校验可被绕过，导致非预期的正式资产改写；
- 运行时脚本可在项目根目录外读取、写入或执行；
- 更新、发布或 bundle 构建链可被注入不可信文件；
- 公共文档鼓励用户暴露 API key、登录 token、模型账单或原始受版权保护文本。

模型输出质量、第三方模型服务中断、平台规则变化和用户自行授予的宿主权限不属于传统安全漏洞，但欢迎在 issue 中按“运行边界”说明。

## 负责任披露

请勿在公开 issue 中提交凭据、私人书稿、完整原文、内网地址或可复现的敏感数据。

请通过仓库维护者可见的私密渠道提交：

1. 问题影响与受影响版本；
2. 最小复现步骤，不附带真实秘密或受版权保护内容；
3. 预期行为与实际行为；
4. 可选的修复建议或补丁。

维护者会先确认影响范围，再安排修复、测试和公开说明。公开说明会描述修复版本与风险边界，不复述敏感细节。

## 用户安全建议

- 不要把 API key、会话 token、代理密码或个人素材写进书籍项目、skill 文档或 prompt。
- 仅在可信项目目录运行写入类 workflow；对导入、批量改写和结构迁移先阅读影响范围。
- 更新 skill 与刷新书籍协作环境是两层动作。刷新环境不会改正文，但结构迁移必须单独确认。
- GitHub 发布前必须使用公开发布脚本和公开审计；不要直接推送开发分支。

## 安全验证

公开发布的最低验证包含：

```bash
node scripts/public-release-audit.js --repo-root . --json
node scripts/production-smoke-matrix.js --repo-root . --json
git diff --check
```

审计失败时不得发布。审计通过也不替代用户对模型权限、第三方服务和真实书稿备份的判断。
