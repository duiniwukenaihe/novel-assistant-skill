# 故事承重合同

本合同约束短篇、长篇、续写、回炉和审阅的故事质量语义。它检查的是故事是否成立，不是文档是否恰好使用某组字段名。

## 一、故事脊柱

进入大纲、卷纲、小节大纲或正文前，至少能回答：

1. 什么事件打破了原有生活？
2. 为什么偏偏是这个主角被迫卷入？
3. 主角为什么不能轻易离开或恢复原状？
4. 哪个选择会产生不可逆后果？

无法回答时，先补故事设计，不用字数、设定说明或新角色掩盖空缺。

## 二、压力系统

故事不要求每项都平均出现，但必须形成可持续压力，不能只靠信息披露推进：

- 外部压力：危险、对手、规则、资源或时间限制。
- 内部压力：欲望、缺陷、恐惧、羞耻、执念或自我欺骗。
- 关系压力：盟友分歧、亲密关系债务、团队摩擦或信任损耗。
- 价值压力：对手代表一套能自圆其说、会迫使主角选择的立场。
- 拖延代价：主角不行动时，局面也会恶化。

设定、能力和世界观只有转化成选择限制、关系变化或现实代价后，才算进入故事。

## 三、揭示必须改变局面

重大真相、反转或伏笔回收至少改变一项：

- 危险程度；
- 人物关系；
- 对人或事件的判断；
- 可选行动及其代价。

只补充解释、不改变任何选择的揭示属于说明材料，不应冒充剧情推进。优先呈现证据和后果，再解释来源。

## 四、最小叙事闭环

每个可写单元都应形成：

`目标 -> 可见阻力 -> 主角选择 -> 后果/代价 -> 关系或判断变化 -> 承接问题`

短篇在“全篇 + 小节”两层使用；长篇在“全书 + 卷/阶段 + 章节”多层使用。章节不必机械填满所有字段，但缺少选择、代价或承接时，不能仅凭字数达标通过故事门。

## 五、工作流边界

- **生产写作**：大纲或写作提要缺少必要承重信息时可以阻断写正文，并给出最小补全清单。
- **只读审阅**：不得因旧大纲字段名不同而拒绝阅读正文。先做语义映射；信息确实缺失时，再把缺口列入修复方案。
- **用户反馈**：讨论中的候选意见先进入待确认设计；确认后的最终方案才回写规划和记忆，并使受影响的写作提要失效。
- **记忆**：只保存已接受事实、已确认设计、未闭合承诺和证据引用；推测、外部模板及模型临时建议不得成为既定事实。
- **篇幅**：不规定固定章节数或固定字数。篇幅服务于当前单元的剧情功能，明显偏离作品基准时检查缺失或重复功能，不机械补删。

## 六、渐进加载

入口只引用本合同。专业模块在当前任务涉及大纲、写作提要、正文故事门或内容审阅时才读取；不得把外部 skill、研究报告或完整模板一并注入运行上下文。
