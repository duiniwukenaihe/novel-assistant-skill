# 拆文到作者吸收

拆文不是复述原书，而是把成功样本拆成“可迁移的情绪技术”。所有可借鉴结论都必须能转成当前项目的写作动作。

## 拆文吸收四层

1. **欲望层**：读者在这一段获得什么补偿？例如被低估后反杀、被偏爱、窥见禁忌、先知优越、强者护短。
2. **结构层**：作者如何让欲望成立？看铺垫、误导、延迟、反应层、代价、回扣。
3. **场景层**：技巧落在哪种场景？开篇压力、对峙、交易、战斗、关系试探、公开审判、章尾悬念。
4. **转译层**：去掉原书人名设定后，换成当前项目可用的角色压力、物件、势力和章节位置。

## 拆文报告必须沉淀 Top 技巧卡

Stage 5 `拆文报告.md` 增加“作者吸收笔记”小节，选 5-8 个真正可迁移的技巧：

```markdown
### 吸收卡：{技巧名}
- 来源章节：{第N章/章节范围}
- 原书功能：{在原书中解决什么情绪或节奏问题}
- 技巧机制：{铺垫→释放→余波 / 信息压制→揭示→反应层 等}
- 可迁移骨架：{功能位 + 情绪链 + 节奏点}
- 写作动作：{换到新书时可用于开场/中段/章尾/卷高潮的具体动作}
- 禁抄替换：{不能照搬的设定、台词、场景；建议替换方向}
```

同时落盘 `作者吸收笔记.md`，便于 `story-long-write` 开书和日更时直接召回。

## 文风吸收边界

`文风.md` 只学习表达技巧：句长、停顿、对话潜台词、视角距离、情绪交替。不得用文风文件替代剧情机制；更不得保留对标书的具体桥段、名词和台词。
