# 短篇平台与投稿契约

平台配置只负责把同一个故事核翻译成目标读者更容易进入的阅读节奏，不替代人物、因果和情感质量。

## 使用顺序

1. 从用户选择、项目设定或素材卡确认目标平台与主题材。
2. 调用 `scripts/short-writing-profile.js`，一次只加载一个平台配置和一张题材卡。
3. 与用户确认开篇承诺、阅读停顿方式和结尾兑现后，再进入节奏套路选择。
4. 目标平台或题材变化时重新生成配置，不叠加旧卡。

## 配置字段

- `target_platform`：用户确认的平台；未确认时保持通用配置。
- `opening_pitch`：开篇需要尽早建立的困境、损失、异常或关系裂口。
- `reading_breakpoint`：按因果和信息变化设置的阅读停顿，不按固定字数机械切割。
- `ending_contract`：结局必须兑现的故事承诺、人物选择与情绪结果。
- `evidence_source`：平台判断来自用户确认、当前扫榜数据、公开方法资料或通用经验。
- `confidence`：`verified`、`reviewed` 或 `unverified`。没有当前数据时不得标成已验证。

## 平台边界

平台偏好会变化。字数、付费节点、热门题材和流量规则只能作为带日期的证据，不能写成永久真理。没有联网或近期数据时，优先保证：开篇有具体问题、每节有状态变化、反转有前因、人物选择有代价、结尾兑现核心承诺。

## 禁止做法

- 不把多张题材卡同时塞进上下文。
- 不为了匹配渠道而牺牲人物逻辑或凭空制造苦难。
- 不复制对标作品的句子、人物名、专有设定或独特桥段。
- 不把“平台喜欢”当作跳过用户确认和质量门的理由。
