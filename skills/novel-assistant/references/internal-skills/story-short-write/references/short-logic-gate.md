# short-logic-gate.md：短篇角色与情节逻辑门

本文件用于短篇构思、生成设定、小节大纲、正文写作和回炉前的硬性自检。它解决的不是标点或 AI 腔，而是“短篇很短却写散、角色性别混乱、关键关系不可信、豪门/权威角色替主角完成主题”的结构性问题。

## 使用时机

- 生成 `设定.md` 前：先建立角色锁定卡。
- 生成 `小节大纲.md` 前：先过情节可行性门。
- 写当前小节前：只读取本节涉及角色的锁定卡、本节因果链、上一节采用锚点和当前 Brief。
- 回炉/重写前：先判断是正文表达问题，还是设定/大纲已坏；如果设定或大纲已坏，先修上游文件，不直接润色正文。

## 角色锁定卡

每个有名字、会推动情节或影响主角选择的角色，必须在 `设定.md` 写成表格。缺一项不得进入正文。

| 字段 | 必填要求 |
|---|---|
| 角色名 | 本名 + 常用称呼，避免同音/同姓混淆 |
| 性别/称谓/视角身份 | 性别、第一人称关系、别人如何叫他/她；例如“林昭：女，第一人称‘我’，母亲叫昭昭，旁人叫林小姐” |
| 年龄/职业 | 至少写年龄段和社会位置 |
| 与主角关系 | 血缘、雇佣、恋爱、债务、利益、旧恩等必须明确 |
| 本篇目标 | 他/她主动想要什么，不得只为主角服务 |
| 行动边界 | 他/她不会做什么，防止剧情强行拉扯 |
| 声口标记 | 1-2 条说话习惯或禁用称呼 |
| 证据物/证据事 | 支撑关系和反转的物件、文件、证人或过去事件 |

### 写作硬规则

- 第一人称短篇中，主角性别、称谓和社会身份一旦确定，全文不得漂移。女性主角不能在叙述里被误写成“儿子”“男方”“娶妻方”等。
- 任何“他/她/儿子/女儿/老板儿子/亲家母”等称谓必须能回指到角色锁定卡。回指不明时，改成角色名或换句。
- 亲属关系和雇佣关系不能靠读者猜。写“老板儿子”“沈少”“远房亲戚的儿子”时，必须明确这是骗局身份、真实身份，还是旁人误认。
- 回炉只改正文时，不得改掉用户确认的角色名和性别；若必须改，先提示这是设定级变更。

## 情节可行性门

所有高价值反转必须过“动机-渠道-证据-代价”四问。答不出就不能写成正文。

| 问题 | 通过标准 |
|---|---|
| 谁主动做了这件事？ | 写出行动者，不用“刚好”“突然”“临时”糊过去 |
| 他/她为什么现在做？ | 有即时动机，且符合角色锁定卡目标 |
| 他/她怎么做到？ | 有可见渠道：人脉、钱、职业权限、信息来源、地理位置 |
| 做这件事要付什么代价？ | 钱、风险、名声、关系破裂、暴露秘密，至少一个 |

### 巧合预算

- 一个完整短篇最多允许 1 个大巧合；这个巧合必须在前文有 2 个以上弱铺垫。
- 如果同一篇同时出现“刚好认识”“刚好是老板儿子”“刚好同一天用同一个包间”“刚好愿意出钱背书”，必须合并、删减或改成主角主动设计。
- 豪门/权威角色可以制造压力或提供镜子，不能直接替主角解决核心压迫。
- 反转的快感必须来自“主角选择 + 前文证据回收”，不能来自“更有钱的人出来宣布主角值钱”。

## 主题防漂移

写 `设定.md` 时必须列出一条“主题承诺”，正文每节都要对齐它。

示例：

- 主题承诺：不是嫁入豪门，而是撕掉别人给我的价签。
- 禁止漂移：沈家不能用 280 万证明主角有价值；沈家只能揭露母亲的算法荒唐，最终决定必须由主角自己做。
- 主角关键行动：主角至少在高潮前完成一次不可替代的主动选择，例如拒绝收钱、公开改写价签、拿出证据反击、自己离开羞辱现场。

自检问题：

- 如果删掉豪门/权威角色，主角是否仍有主动行动？
- 高潮是不是别人替主角讲话？如果是，必须增加主角自己的决定或反击动作。
- 钱、身份、学历、婚恋对象是否被写成主角价值的唯一证明？如果是，重写价值落点。

## 丰满度门

每节不只要“发生事件”，还要有可读的场面。写正文前，小节大纲每节至少满足：

- 3-5 个子事件，每个子事件写清“谁做了什么 + 情绪/信息功能”。
- 至少 2 个具体物件或环境细节，但必须服务情节，不能纯摆设。
- 至少 1 个角色选择：主角或关键配角做出会改变后续的动作。
- 至少 1 个后果：这一节结束后，关系、证据、公众判断、风险或主角心理边界发生变化。
- 不得只有“见面、对话、反转、打脸”四个标签；标签必须落成具体动作。

## 失败处理

命中以下任一项，停止正文写作，回到设定/大纲修复：

- 角色性别、亲属称谓、视角身份混乱。
- 关键反转只能靠强巧合成立。
- 主题从自我价值漂到嫁豪门、被豪门认可、被男人/权威定价。
- 每节只有剧情标签，没有具体子事件和后果。
- 高潮由外部权威完成，主角没有不可替代行动。

修复顺序：

1. 改 `设定.md` 的角色锁定卡和主题承诺。
2. 改 `小节大纲.md` 的因果链、巧合预算和每节子事件。
3. 再重写对应正文小节。
4. 最后才做去 AI 味、标点和句式精修。
