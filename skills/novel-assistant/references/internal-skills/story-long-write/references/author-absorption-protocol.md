# 作者式吸收协议

用于扫榜、拆文、对标召回、开书和日更写章。目标不是机械摘字段，而是像成熟作者一样，把外部样本转译成自己的题材判断、剧情策略和正文动作。

## 核心原则

1. **先读读者欲望，再读技巧**：榜单和拆文里的技巧只是在服务某种欲望，例如被低估后的反杀、关系确认、禁忌窥见、弱者翻盘、强者护短。说不清读者欲望时，不得直接照搬桥段。
2. **学习情绪逻辑，不学习表层事件**：可迁移的是“压抑如何蓄势、信息如何延迟、反应层如何放大、代价如何兑现”，不是具体角色、门派、职业、台词和场面。
3. **技巧必须落到写作动作**：每条吸收结论都要转成可执行动作：用在哪一卷、哪几章、哪个角色压力、哪个场景、哪个章尾钩子。
4. **反抄袭先于借鉴**：任何样本技巧都必须写出“不可照搬项”和“替换策略”。没有替换策略的对标，不进入正文 prompt。
5. **作者判断大于数据堆砌**：扫榜给市场证据，拆文给结构证据，但最终要判断“我这本书凭什么成立”。不要把榜单热词拼贴成书。

## 五步吸收法

每次从扫榜、拆文或对标书学习时，按以下五步输出：

| 步骤 | 要回答的问题 | 输出 |
|---|---|---|
| 1. 读者欲望 | 读者追它，是想获得什么情绪补偿？ | 欲望词 + 触发场景 |
| 2. 技巧机制 | 作者用了什么结构让这个欲望成立？ | 铺垫、误导、反应层、延迟、回扣、代价 |
| 3. 可迁移骨架 | 去掉人名设定后，剩下什么可复用骨架？ | 功能位 + 情绪链 + 节奏点 |
| 4. 本书转译 | 换到当前项目，应该由谁、在什么场景里承担？ | 角色/势力/物件/章节位置 |
| 5. 禁抄替换 | 哪些具体元素不能碰？怎么替换？ | 不可照搬项 + 替换方案 |

## 作者吸收卡

扫榜、拆文、开书和写章都可以使用以下格式。每张卡只记录一个真正可用的技巧，宁少勿滥。

```markdown
### 吸收卡：{技巧名}
- 来源：{榜单/书名/章节/文件路径}
- 读者欲望：{读者想获得的情绪补偿}
- 技巧机制：{作者如何铺垫、延迟、释放、放大}
- 可迁移骨架：{功能位 + 情绪链 + 节奏点}
- 本书转译：{用于第几卷/第几章/哪个角色/哪个场景}
- 正文动作：{开场压力 / 中段误导 / 爽点释放 / 章尾钩子等具体写法}
- 禁抄替换：{不能照搬的具体设定、台词、场景；替换成什么}
- 风险：{同质化、节奏不适配、人物动机不够、平台不适配等}
```

## 不合格吸收

- 只写“节奏快、爽点密、反转强”，没有说明怎么快、爽在哪里、反转靠什么信息差。
- 把榜单热词拼成设定，例如“御兽 + 系统 + 护短 + 杀疯”，但没有主线压力和情绪承诺。
- 拿对标书文风当万能答案，忽略本章细纲、角色状态和章节契约。
- 只抽象总结，不落到章节动作。
- 只记录“可借鉴”，不记录“不可照搬”。

## 写作接入点

- **扫榜后**：`选题决策.md` 每个推荐选题至少附 1-2 张吸收卡，说明市场样本能转成什么写法。
- **拆文后**：`拆文报告.md` 或 `作者吸收笔记.md` 必须沉淀 Top 技巧卡，优先选择能支撑本项目的技巧。
- **开书时**：从吸收卡中选择 3-5 张写入 `设定/作者吸收笔记.md`，作为本书的创作方法库。
- **写章前**：只召回与本章目标情绪相关的 1-2 张卡，把“正文动作”写入本章写作意图；不相关的卡不进 prompt。
- **回炉时**：若用户指出进度过快、桥段太早、AI 味强，先回看吸收卡是否误用：是技巧转译失败，还是章节位置不对。
