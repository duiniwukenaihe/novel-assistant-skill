# 新书启动工作流

用于 `/novel-assistant` 收到「开书 / 新书启动 / 我想写小说 / 帮我写书 / 写长篇」时。目标是把用户从灵感带到可控生产：先确认市场与情绪承诺，再确认人物、剧情引擎、总纲、前 10 章细纲，最后才进入正文。

**总规则：禁止裸写正文。** 没有通过下列门禁前，不写第一章正文；最多只能写概念、设定、大纲、细纲和 Chapter Contract。

## Project Readiness Gate

检查项目根：

1. `.story-deployed` 不存在：先执行内部模块 `story-setup` 部署基础设施；部署完成后继续本文件，不要停在 setup。对用户提示统一写 `/novel-assistant 准备写书`。
2. `.active-book` 存在：读取当前书名；如果用户明确要开新书，创建或切换到新书目录。
3. 已有 `正文/` + `追踪/` 且用户只说「继续」：退出本 workflow，改走 `workflow-daily.md`。

通过后确认写作工作目录，并准备以下基础目录：`设定/`、`大纲/`、`追踪/`、`正文/`、`对标/`。

## Market Positioning Gate

一次只问最关键问题，优先复用 `选题决策.md`：

1. 目标平台：起点 / 番茄 / 晋江 / 七猫 / 其他。
2. 目标读者情绪：爽感释放 / 意难平 / 期待感 / 紧张好奇 / 治愈 / 反转震撼。
3. 题材赛道：主类型 + 副类型。
4. 对标对象：已有对标书、拆文库，或需要先扫榜。
5. 差异化承诺：同赛道里这本书的新鲜点。

产物：

- `设定/题材定位.md`：平台、读者、目标情绪、题材赛道、对标书、差异化承诺、失败风险。
- 如用户没有方向且要求市场数据：内部路由到 `story-long-scan`，生成或导入 `选题决策.md` 后再回来。

## Author Absorption Gate

加载 [author-absorption-protocol.md](author-absorption-protocol.md)。在正式设计核心设定前，先把扫榜、拆文、本地拆书或用户指定对标转成作者能用的吸收卡。

输入来源按优先级读取：

1. 项目根 `选题决策.md` 里的「作者吸收卡」。
2. `对标/{书名}/作者吸收笔记.md` 或 `拆文库/{书名}/作者吸收笔记.md`。
3. `对标/{书名}/拆文报告.md`、`剧情/情绪模块.md`、`剧情/节奏.md`、`文风.md` 中可转译内容。
4. 用户本次口述的喜欢/不喜欢的书和桥段。

筛选规则：

- 只选 3-5 张最贴合本书目标情绪和作者优势的吸收卡。
- 每张卡必须包含：读者欲望、技巧机制、可迁移骨架、本书转译、正文动作、禁抄替换。
- 只写“节奏快/爽点多/文风好”的卡不合格，必须重写为可执行动作。
- 如果没有可用吸收卡，不阻塞开书，但要在 `设定/作者吸收笔记.md` 写明「暂无外部吸收，当前按原创设定推进，建议后续扫榜/拆文补强」。

产物：

- `设定/作者吸收笔记.md`：本书创作方法库。后续写章只召回与当前章节目标情绪相关的 1-2 张卡。

## Core Promise Gate

把市场方向压成能写的故事承诺：

1. 一句话梗概：主角 + 目标 + 阻碍 + 反转。
2. 黄金三章承诺：第一章钩子、第三章前必须兑现的第一个爽点或反转。
3. 长线阅读理由：读者为什么愿意追到第 30 章、第 100 章。

产物：

- `设定/核心设定.md`
- `追踪/主线承诺.md`

## Character Relationship Gate

先立可长期驱动剧情的人，再立关系压力：

1. 主角：底层欲望、当前目标、能力/金手指、缺陷、行为红线、语言边界。
2. 对手：想要什么、为什么挡主角、何时升级。
3. 核心配角 2-3 个：功能位、利益关系、情绪功能。
4. 关系压力：谁欠谁、谁骗谁、谁需要谁、谁不能说破。

产物：

- `设定/关系.md`
- `设定/角色不变量.md`：角色不变量索引。
- `设定/角色不变量/{角色名}.md`：每个核心角色的底层欲望、当前阶段目标、行为红线、认知边界、语言边界。
- `追踪/角色状态.md`

## Plot Engine Gate

设计能滚动生产几十章的剧情发动机：

1. 世界/行业/力量规则：什么规则制造冲突。
2. 重复爽点循环：每 3-5 章如何制造期待、推进、兑现。
3. 主线压力：主角不行动会失去什么。
4. 伏笔债：前 10 章要埋哪些、何时回收。

产物：

- `设定/世界观/核心规则.md`
- `设定/剧情引擎.md`
- `追踪/伏笔.md`

## Macro Outline Gate

先搭全书鸟瞰，再细化第一卷：

1. 全书卷级结构：每卷功能、字数、核心事件、状态变化。
2. 第一卷目标：第一个大爽点、阶段反派、阶段关系变化。
3. 付费/追读节点：第 1、3、10、20、30 章的期待点。

产物：

- `大纲/大纲.md`
- `大纲/第1卷/卷纲.md`
- 旧项目兼容路径：`大纲/卷纲_第1卷.md`
- `追踪/时间线.md`

## Detailed Outline Gate

默认先做前 10 章细纲，不强行一次生成全书每章。每章细纲必须包含：

1. 核心事件：谁做了什么。
2. 情节点序列：常规 3000 字章节 10-15 个情节点。
3. 目标情绪：本章交付什么。
4. 章首钩子、爽点、章尾钩子。
5. 新增/推进/回收的伏笔。
6. 字数目标。

产物：

- `大纲/细纲_第001章.md` 到 `大纲/细纲_第010章.md`
- 新项目写入 `大纲/第1卷/细纲_第001章.md` 到 `大纲/第1卷/细纲_第010章.md`；旧项目由内部模块 `story-setup` 迁移或读取兼容。
- `追踪/章节索引.tsv`（50+ 章项目后续持续维护）

## Production Entry Gate

进入正文前生成当前章节契约：

1. 读取 `大纲/第1卷/细纲_第001章.md`（旧项目回退 `大纲/细纲_第001章.md`）、角色不变量、主线承诺、伏笔表。
2. 生成 `追踪/当前章节契约.md`，明确本章必须交付的情节点、情绪、钩子、伏笔动作和禁止漂移项。
3. 如项目已有 v0.8/v0.9 脚本，运行：

```bash
node scripts/story-schema-build.js <book-project-dir> --write
node scripts/current-contract-build.js <book-project-dir> --chapter 1 --write
node scripts/oh-story-doctor.js <book-project-dir> --mode draft --write
```

`gate.status=fail` 时先修复；`warn` 时写入风险说明；`pass` 后才进入 Phase 4 正文写作。

## Handoff

完成本 workflow 后，向用户汇报：

- 已确认的题材/平台/目标情绪。
- 已选作者吸收卡和它们会落到哪些卷/章节。
- 核心人物和剧情引擎。
- 已落盘的大纲和前 10 章细纲。
- 第 1 章 Chapter Contract 路径。
- 下一步：进入 `Chapter Contract -> 正文写作 -> Plot Drift Gate -> State Delta Ledger -> Chapter Handoff Pack`。
