# state-delta-ledger.md：状态变化账本

State Delta Ledger 记录本章对后续写作有影响的变化。它不是章节摘要，只写“状态发生了什么变化”。

## 写入时机

- Plot Drift Gate 通过后立即写。
- 回炉、重写、改纲后重新写。
- 批量日更时每章都写，不等本批全部写完。

## 建议写入位置

| 文件 | 写入内容 |
|---|---|
| `追踪/上下文.md` | 最近 3-5 章 State Delta 摘要、最后完成章节、下一章注意事项 |
| `追踪/角色状态.md` | 角色身份、能力、关系、公众形象、伤势、资源变化 |
| `追踪/伏笔.md` | 伏笔新增、推进、误导、半兑现、回收 |
| `追踪/时间线.md` | 故事时间、事件顺序、地点移动 |

## 模板

```md
## 第 N 章 State Delta

### 角色变化
| 角色 | 原状态 | 新状态 | 证据 |
|---|---|---|---|

### 关系变化
| 关系 | 变化 | 后续影响 |
|---|---|---|

### 资源/能力变化
| 对象 | 变化 | 限制 |
|---|---|---|

### 伏笔变化
| 伏笔 | 状态 | 位置 | 后续动作 |
|---|---|---|---|

### 新增事实
| 类型 | 内容 | 是否已入设定 |
|---|---|---|

### 未解决问题
- 
```

## 压缩规则

- `追踪/上下文.md` 只保留最近 3-5 章 State Delta。
- 长期状态必须同步到角色、伏笔、时间线或设定文件。
- 如果正文没有产生某类变化，该小节写“无”，不要杜撰变化。
