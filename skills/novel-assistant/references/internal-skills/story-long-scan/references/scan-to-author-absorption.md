# 扫榜到作者吸收

扫榜不是把热词拼成书名，而是判断“哪些读者欲望正在被反复满足”，再决定作者能不能用自己的素材重新交付。

## 扫榜吸收四问

1. **重复信号是什么**：同平台、同题材、相近榜单里，反复出现的是设定词、人设关系、开篇压力，还是爽点结构？
2. **读者买账的欲望是什么**：护短、逆袭、低谷翻盘、身份反转、关系确认、禁忌感、强者装弱、经营成长，哪一个最核心？
3. **新作者可复用的技巧是什么**：开篇压力、信息差、反应层、升级循环、金手指反馈、关系拉扯，哪一个可被当前作者掌控？
4. **不能照搬什么**：榜上书的具体门派、职业、系统名、台词、标志场面和书名结构不得机械复制。

## 选题决策必须增加作者吸收卡

每个推荐选题至少写 1 张吸收卡；排名最高的推荐选题写 2 张。格式：

```markdown
#### 作者吸收卡：{技巧名}
- 市场信号：{来自哪些榜单/样本的重复现象}
- 读者欲望：{读者追这个方向想获得什么}
- 可迁移技巧：{结构/节奏/反应层/关系压力，不写表层设定}
- 本书转译：{如果开这本书，怎么换成自己的主角、冲突和前三章动作}
- 禁抄提醒：{不能照搬的书名词、设定、桥段、台词}
- 验证动作：{黄金三章或拆文需要验证什么}
```

## 结论语气

输出要像作者给自己做选题会，而不是数据分析周报。可以保留数据证据，但每个结论都要回答：“所以我该怎么写，先写哪三章来验证？”
