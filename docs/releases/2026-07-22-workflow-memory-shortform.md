# 2026-07-22 Workflow / Memory 与短篇生产内核

## 发布目标

本轮优先稳定单个短篇项目的完整生产生命周期。目标不是增加更多提示词，而是保证规划、逐节写作、质量验收、正式接受、记忆更新和跨会话恢复使用同一套任务事实源。

## Workflow 成果

- `workflow_id` 对应的持久任务目录是任务权威；界面焦点不再冒充完整任务。
- 短篇按全篇规划和已确认总节数推进，最后一节后进入组装，不生成计划外小节。
- 每节按写作提要、正文、机器门、故事门、接受提交和下一节写作提要形成闭环。
- 用户反馈先做影响分析：表达问题留在当前节，因果/人物/反转问题回写规划，结构变化回到总节数锁定。
- 业务停靠使用可见状态和数字候选；真正的参数、路径、权限和进程故障才使用非零退出码。

## Memory 成果

- 任务、故事记忆、制品和用户偏好通过 repository 访问，底层存储可替换。
- 默认本地 adapter 保证 skill 单独安装即可工作；外部工作台未来可以通过稳定契约接管 PostgreSQL 权威存储。
- 当前小节只读取已接受事实、活跃人物、未闭合承诺、连续性义务和声口规则。
- 记忆读取回执绑定目标小节与快照版本。相关事实变化只让相关证据失效，不触发无条件全篇重跑。
- 污染输出、未确认事实和非主任务分支不会进入当前记忆。

## 公开版本边界

- GitHub 包含公共短篇模块、公共 Workflow/Memory 内核和必要运行脚本。
- 本地可选扩展、个人素材、原始文本、运行日志、内部地址与私有路由均由构建审计移除。
- 当前先声明“短篇生产链优先稳定化”。长篇和其他模块继续可用，但不会被描述成已经完成同等级别的生产验收。

## 验证原则

确定性测试验证状态机、存储契约、记忆快照、路由和公开构建边界。真实 Claude Code、Codex、ZCode 行为验收单独记录；没有真实宿主证据时，不用 fake host 或单元测试替代该结论。
