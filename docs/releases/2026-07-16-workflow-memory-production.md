# 2026-07-16 工作流、记忆与行为验收整合说明

本说明记录本次 GitLab `main` 相对上一轮发布的生产化整合。它覆盖写作工作流、长期记忆、章节提交、三端行为验收和公开发布隔离；不包含用户正文、私有素材或私有短篇资产。

## 本轮解决的问题

### 任务与多会话

- `current-task.json` 降级为界面焦点，不再作为任务事实来源。
- 每个任务都以 `workflow_id -> 追踪/workflow/tasks/<workflow_id>/task.json` 保存不可变快照；提交、结果包和记忆投影按任务快照归属。
- 同一目标的恢复、重规划和多会话分支归入 `task_family_id`。收件箱以任务族计数，暂停或被替代的分支不再重复显示为多项未完成任务。
- 会话写入使用书籍级租约。多个宿主可以只读查看；写作和结构修改只能由一个明确持有租约的会话推进。失活任务进入“待认领”，不静默抢占仍活跃的会话。

### 正式资产与章节提交

- 正文、伏笔、时间线、角色状态、上下文和交接包改为“候选暂存 -> 质量门 -> `chapter-commit prepare/accept` -> 投影”的提交链。
- `accept` 前检查目标文件是否被并发修改；中途失败回滚备份；投影可通过 `replay` 重放。
- 运行时会审计未登记的正式资产写入。候选稿、临时修复脚本和不可信审阅材料不能直接被当作正式事实。

### 记忆与上下文

- 记忆条目带来源类型：`legacy`、`canonical`、`user_confirmed`。只有旧项目迁移资产可自动刷新；正式记忆保留章节提交、任务族和版本链。
- 日更后的受影响记忆会按提交增量投影；来源变化先隔离或重建，不再因追踪文件整文件哈希变化阻塞整本书。
- 上下文装配排除暂停或失效分支的记忆，并结合实体别名、中文 n-gram/BM25 和证据位置召回，降低跨卷承诺与人物别名漏检。

### 运行、退化与 token

- runner 托管模式才具备无人值守能力：可管理子进程、心跳、重试预算、断点和流式健康监控。
- 用户在既有 Claude Code/Codex/ZCode 会话中直接调用 skill 属于协作模式：保留任务、结果包和断点，但不会伪称能拦截宿主隐藏思考或 API 流。
- 输出健康门会阻止重复 n-gram、工程词泄漏、格式错位和明显污染进入正式资产；发生异常时保留最后可信断点，缩小粒度后最多受控重试一次。
- 行为验收仅记录宿主实际返回的输入、输出、缓存 token 和耗时，不从 token 推导美元费用。Codex 隔离验收遇到连续网络重连会快速结束为宿主不可用，避免长期空转。
- ZCode CLI 未在 stdout 提供 usage 时，会只读查询本机已完成的同一 `session_id + trace_id` 遥测记录；无可用遥测时如实标记，而不是伪造成本。

### 专业 workflow 与上游反哺

- 长篇裸调用默认停靠在题材、总纲、卷纲、阶段细纲和 Brief；细纲不足时返回 `outline_underfilled`，禁止为凑字数自造剧情。
- 范围审阅按作品层、卷层、阶段细纲层、章节层组织。证据批次和 Agent 数量只作为内部参数，不替代完整审阅生命周期。
- 选择性吸收上游 `bf70c26` 的“细纲不足停靠、单轮正文上限、题材正文卡、长篇节奏门”等原则；题材卡按需加载一张，不整库注入上下文。
- 公开短篇保持使用公开 `story-short-write`；私有扩展只存在于本地私有安装，不参与公开发布包。

## 当前生产边界

本版本适合个人作者的交互式生产、可恢复的长任务和受控 runner 托管任务。它不宣称在所有 Claude/Codex/ZCode 网络、插件和模型条件下都能完全无人值守。正式发布候选仍需运行真实宿主行为矩阵，至少覆盖路由、指定节写作、任务恢复、退化中止和多会话写锁。
