# Workflow / Memory 全局生产架构计划

## 目标

`novel-assistant` 的所有专业流程共用同一套控制面与记忆面。短篇、长篇、审阅、拆文、扫榜、去 AI、导入和 setup 可以有不同阶段，但不得各自发明任务身份、记忆写入、正式资产提交或恢复规则。

## 全局不变量

1. **任务真相唯一**：`tasks/<workflow_id>/task.json` 是任务权威；`current-task.json` 只表示界面焦点。
2. **阶段所有权唯一**：每个阶段只有一个 `owner_module`；专业子 skill 只能返回 Result Packet，不能推进任务。
3. **上下文版本唯一**：一个阶段只消费一个被登记的 Memory Contract/阶段包；托管运行必须回显读取回执。
4. **正式写入唯一**：创作资产先写暂存区，经质量门后通过事务提交；任何直接改正式资产的行为在当前阶段立即阻断。
5. **记忆生效滞后于事实**：聊天先进入工作记忆；确认方案先回写规划资产；正文采用后才投影作品事实。
6. **完成必须无债务**：未投影的已确认方案、运行中的修订队列、未接受的正式写入、记忆冲突或投影债务存在时，workflow 不得完成。
7. **多任务不猜焦点**：写入命令在多个未完成任务并存时必须携带 `workflow_id`；UI 焦点不能成为提交权限。
8. **公私有共享内核**：私有短篇只增强资讯、素材血缘、学习和个性化，不复制或绕过公开生产状态机。
9. **本地/PG 单主**：当前本地 repository 与未来 novel-project PostgreSQL 使用同一 identity/revision/outbox 契约；同一时刻只有一个 authority。
10. **可见交互唯一**：所有专业模块只返回阶段产物和 Result Packet；全局状态机独占“自动续跑还是停靠作者选择”的决定权。安全内部阶段自动续跑，作者决策点统一返回最多四项数字菜单。
11. **已确认阶段原子收束**：选择已确认后，读取、暂存写入、阶段完成命令和结果包接受必须同轮完成；暂存产物存在但结果包缺失时不得产生普通可见回复。
12. **审计范围分层**：阶段接受只核验本阶段声明的正式资产，任务完成再核验全任务资产；无关历史漂移不能阻断当前阶段，未受控写入也不能逃过最终审计。
13. **活动任务必须可推进**：状态摘要不是工作流终点。单个焦点任务展开后必须直接投影当前阶段的 `1/2/3/4` 动作；多个任务才先选择任务，激活后立即投影该任务动作。任何非终态页面都必须包含可执行迁移、暂停或自由输入入口。
14. **旧协议只迁移、不降级**：支持的上游或旧版 `novel-assistant` 长篇先保留正文与证据，再把任务身份、阶段和回执迁移到当前协议。旧回执只能作为历史证据；缺少当前质量证据的阶段必须重验，不能通过放宽 v2 校验继续推进。
15. **运行阶段必须有执行合同**：必需阶段进入 `running` 前，必须已经拥有确定性完成命令、已装配的上下文包或明确的作者决策合同。三者皆无时进入 `contract_blocked` 的四项恢复菜单，禁止显示“正在执行”或重复读取不存在的上下文包。
16. **迁移通过继任任务完成**：旧任务与旧证据保持不可变历史；迁移创建当前协议继任任务，焦点转向继任任务，并从最早没有可信验收的生命周期节点继续。仅“文件存在”只能得到 `needs_review`，不得继承为 `accepted`。

## 仍由宿主承担

- Claude Code / Codex / ZCode 已存在的交互会话无法被纯 skill 强制中断隐藏 thinking。只有 `workflow-runner` 托管子进程具备流式退化早停、重试预算和租约。
- 真实 token usage 取决于宿主是否返回 telemetry；缺失时只记录“不可用”，不伪造费用。
- PostgreSQL 表、连接池、鉴权和 adapter 实现属于 novel-project；skill 不保存数据库凭据。

## 发布验收

每次改动 workflow/memory 后只跑受影响的确定性测试：模板 owner、Memory Contract 回执、正式资产事务、任务恢复、多任务写入绑定、公私有内核一致性。真实付费模型行为矩阵作为发布候选门，由作者按需执行，不在日常修复中自动消耗 token。
