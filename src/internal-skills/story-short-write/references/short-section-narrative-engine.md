# 短篇小节故事引擎门

本门解决「大纲没漏事，但正文仍然不好看」。信息顺序正确不等于剧情成立；查表、看文件、读邮件、解释记录都只是信息载体。

## 每节必有

1. **承接上节**：第 2 节起必须接住上一节留下的问题，不另起一条方便写的新调查线。
2. **场景动作**：人物在当场做了什么，不是作者摘要。
3. **可见阻力**：谁或什么正在阻止主角。
4. **角色选择**：主角做出会改变后续的行动。
5. **本节兑现**：读者在本节真正得到了哪个答案、反转或判断变化。
6. **关系变化**：人物之间的信任、权力、距离或误解与本节前不同。
7. **代价升级**：本节结束时，下一步比本节开始更难。
8. **压力变化**：本节必须升高、反转或释放压力，形成可感知的起伏。

连续三个子事件都是查资料、核对文件或解释记录，判定为 `live_scene_collision` 缺失。必须把其中至少一个改为人物碰撞、阻拦、交换、背叛或不可撤回的选择。

相邻小节还必须通过钩子链检查：上一节留下的问题在下一节被人物行动承接，本节兑现一件事并生成新的未决问题。运行时为每个节尾钩子生成稳定锚点 `H001/H002...`，用于确认 Brief、正文和下一节承接的是同一个剧情债；这些编号由工具维护，不要求作者手工填写。重复同一发现、同一兑现、同一章末悬念，或下一节绕开旧钩子另起调查线，视为剧情停滞。

人工修改小节只获得“禁止静默覆盖”的写入保护，不获得质量豁免，也不把问题降级为提示。它仍按同一标准参与人物、因果、前后钩子、压力变化和最终全篇收束复验；不通过就明确阻断后续生产，是否重写由作者在看到问题后决定。

## 位置职责

- **开篇**：明确入场钩子和整篇故事承诺。
- **中段**：每节至少改变一次判断或关系，不能只给新资料。
- **高潮**：主角以决定性行动兑现核心爆点，并立即付出代价。
- **结尾**：落实现实后果、责任分配、关系收束和主题回扣，不用一个温情物件代替所有后果。

## 三道契约

1. `小节大纲.md` 为每节生成带稳定 ID 的故事合同。
2. `写作Brief_第NNN节.md` 必须逐项映射这些 ID，不得换掉高潮、动机或后果。
3. 故事质量证据必须为每个 ID 引用正文原句；原句不存在就不能通过。

任何一道失败，回到最近的权威层修复。不允许正文临场补大纲，也不允许事后修改大纲去迎合已写正文。
