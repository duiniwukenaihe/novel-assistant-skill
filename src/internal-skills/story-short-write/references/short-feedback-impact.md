# 短篇反馈影响链

用户在 chat 中提出修改，或直接粘贴人工改稿时，先判断语义影响，再决定回写顺序。改动字数不是判断依据；一句话也可能改变人物动机和后续反转。

## 四级分流

| 级别 | 判断标准 | 正确顺序 | 不改什么 |
|---|---|---|---|
| 表达层 | 措辞、标点、句式、局部外貌或动作质感；不改变事实、因果、人物意图、节尾钩子 | 修当前节候选稿 -> 机器门 -> 故事价值门 -> 重新采用 | Brief、设定、小节大纲 |
| 当前 Brief 层 | 调整当前节动作、对白或信息出现顺序；本节功能、关键事实、钩子和后续不变 | 使旧 Brief 失效 -> 重建当前 Brief -> 重写当前节 -> 双门复检 -> 重新采用 | 全篇设定和小节总结构 |
| 规划层 | 人物动机、关系、关键事实、因果渠道、核心反转、爽点、标题承诺、节奏或后续承接变化 | 先改最小权威上游（素材卡/设定/小节大纲）-> 使受影响 Brief 失效 -> 规划验收 -> 重建当前 Brief -> 重写受影响正文 -> 标记后续复检 | 未被证明失效的已采用好段落 |
| 结构层 | 扩容、缩容、插节、删节、合并、拆分、前后移或重排 | 回到总小节锁定 -> 结构影响审计 -> 更新小节大纲和映射 -> 重建受影响 Brief -> 按范围重写/复检 | 不受影响的小节身份和可复用正文 |

## 硬边界

1. **禁止正文先行、上游事后圆账**：只要反馈改变语义事实或后续承接，就必须先改权威上游，再生成正文。
2. **人工改稿先保留为候选**：先记录来源和原文，完成语义影响检查后才能接受为 canonical；不得被模型静默覆盖，也不得据此偷偷反改设定。
3. **已采用小节重新验收**：任何使正文失效的反馈，都要重新经过机器门、故事价值门和采用锚点；不能只改文件便宣布完成。
4. **下游只失效，不默认全重写**：规划或结构变化先给出受影响 Brief、已写小节、钩子和结尾清单；只重写被证明受影响的范围。
5. **不确定时先只读分析**：`feedback_impact_sync` 只输出影响层级、保留项、失效项、重算项和回写顺序。确认后才进入 `feedback_apply_patch`。
6. **公有/私有同语义**：公开短篇使用 `section_brief`；私有增强可映射为 `first_section_brief/next_section_brief`，但不得改变四级分流和验收顺序。

## 长篇映射

同一原则适用于长篇，只替换权威层级：

`总纲/卷纲 -> 细纲 -> Chapter Contract/Brief -> 正文 -> State Delta/Handoff`

- 纯表达修改只修当前章并复检。
- 当前章局部故事变化先改 Chapter Contract；改变章节功能则先改细纲。
- 影响卷目标、跨章钩子或人物弧线时先改卷纲/细纲和追踪资产。
- 扩容、缩容、插章、合并或重排必须先做 Revision Impact Analysis 和版本快照。
