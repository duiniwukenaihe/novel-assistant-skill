# character-invariants.md：角色不变量

角色不变量用于控制长篇连载中的动机漂移、认知泄漏和口吻突变。它补充角色档案，不替代角色档案。

## 何时建立

- Phase 2 核心设定完成后，为主角、反派、核心配角建立。
- 新角色承担连续剧情功能时建立。
- 角色经历重大转折后更新。

## 写入位置

- 推荐目录：`设定/角色不变量/{角色名}.md`。
- 角色档案 `设定/角色/{角色名}.md` 记录人物基础信息；角色不变量只记录长篇稳定性边界。
- 如果项目尚未拆分目录，可临时写在角色档案的“角色不变量”小节，但 `Plot Drift Gate` 必须能明确读取到这些字段。

## 模板

```md
## 角色不变量：{角色名}

### 底层欲望
- 

### 当前阶段目标
- 

### 行为红线
- 不会：
- 除非：

### 认知边界
- 当前知道：
- 当前不知道：
- 不能提前知道：

### 语言边界
- 常用表达：
- 不会使用的表达：

### 可变化区域
- 可成长：
- 可误判：
- 可被迫妥协：
```

## 检查规则

| 风险 | 错误码 |
|---|---|
| 行动不符合底层欲望或当前目标 | `Motivation_Drift` |
| 角色知道了不该知道的信息 | `Knowledge_Leak` |
| 角色越过行为红线但没有转折铺垫 | `Motivation_Drift` |
| 语言风格突然变化且非剧情所需 | `Motivation_Drift` |

## 多角色 / 多 POV 检查

章节正文可以用显式 POV 标记限定后续段落的角色视角：

```md
## POV：江临

江临只知道任务文本有异常。

## POV：沈岚

沈岚根据自己的线索做出判断。
```

`chapter-stability-check.sh` 会按角色生成扫描视野：

- 没有 POV 标记的旧章节，仍按全章检查所有角色不变量。
- 出现 `POV：角色名` 或 `视角：角色名` 后，只把该 POV 段落纳入对应角色的不变量检查。
- POV 标记前的公共叙述会被所有角色视为公共上下文。
- 某角色 POV 中出现另一角色的禁知词，不自动判定为后者泄漏；必须出现在后者自己的 POV 或公共叙述中才触发。
- 匹配当前 POV 角色的 `不能提前知道`、`不会` 仍分别输出 `Knowledge_Leak`、`Motivation_Drift`。

## 更新规则

- 角色成长可以改变当前阶段目标，不能无说明地改写底层欲望。
- 认知边界只能因为正文中的事件、对话、调查、传承、误导而变化。
- 更新不变量时，必须在 State Delta 中记录原因。
