# chapter-index.md：长篇章节索引

章节索引用于 50+ / 100+ 章项目的稳定性检查与上下文加载。它把章节号映射到正文路径，避免每个 gate 反复扫描 `正文/`。

## 写入位置

```text
追踪/章节索引.tsv
```

字段为：

```text
chapter	title	path	chars
```

示例：

```text
001	第001章 拒绝错误任务	正文/第001章_拒绝错误任务.md	428
002	第002章 追查旧账号	正文/卷一/第002章_追查旧账号.md	512
```

## 自动化入口

```bash
bash scripts/chapter-index-build.sh <book-dir>
bash scripts/chapter-index-build.sh --write <book-dir>
```

`--write` 会刷新 `追踪/章节索引.tsv`。`longform-daily-stability-audit.sh --write` 会在批量检查开始前自动刷新一次索引。

## 使用规则

1. 单章稳定性检查优先读取 `追踪/章节索引.tsv` 查找正文路径。
2. 索引不存在时，脚本保持旧 fallback：读取 `正文/第{N}章_*.md` 或 `正文/第{N}章.md`。
3. 正文目录出现卷内子目录、归档目录或非标准标题时，必须先刷新索引。
4. 每完成 50 章、卷结束、章节移动、回炉改名后，刷新一次章节索引。
5. 索引只记录正文路径和字数，不替代 `追踪/上下文.md`、伏笔表、时间线或角色状态。

## 性能边界

- 适合纯 Markdown 项目，保持可读、可复制、可手动修正。
- 不引入数据库、RAG 或二进制缓存。
- 对 100+ 章项目，索引刷新是一次性目录扫描；后续章节 gate 按章节号直接定位正文。
