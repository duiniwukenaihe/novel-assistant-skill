# 用户个人风格学习协议

本协议用于把用户的长期写作偏好、修改意见和已确认正文沉淀为可复用的作者风格资产。它学习的是“用户自己的判断”，不是对标书文风，也不是模型临时总结。

## 核心边界

1. **文件记忆优先**：长期学习必须落盘到 `设定/作者风格/` 与 `追踪/schema/`，不能只留在聊天上下文。
2. **style-learner 只整理，不创作**：`style-learner` 负责从用户原话、diff、已确认章节和审阅反馈中提炼规则；不直接改正文、不替代 narrative-writer。
3. **硬约束可门禁，软偏好进 prompt**：能机械检测的规则可进入脚本/正文门禁；审美判断只进入写作前 `user_style_constraints`。
4. **单书偏好不污染全局**：本书节奏、人物、势力登场顺序写入当前书项目；跨书稳定偏好才写入全局作者风格。
5. **证据化学习**：每条规则必须有来源证据，至少包含用户原话摘要、文件路径或修改前后差异说明。

## 触发语

用户出现以下表达时，路由到本协议：

- “记住我的风格”
- “以后不要这样写”
- “我喜欢这个版本”
- “按我修改后的风格学习”
- “从我改过的章节里总结我的偏好”
- “这个写法以后保持”
- “这种写法我不接受”
- “这种报告污染以后不要出现”
- “修复方案里不要再重复填充这种词”

## 风格资产结构

```text
设定/作者风格/
  我的写作偏好.md        # 人类可读总览：硬约束、软偏好、单书偏好、示例
  正文风格画像.md        # 句长、段落、对话、标点、心理描写、爽点节奏
  禁用表达.md            # 用户明确不接受的表达、句式、符号、AI味
  优秀样章.md            # 用户确认满意的正文片段索引，不复制整章
  修改偏好案例.md        # 修改前 -> 修改后 -> 学到的偏好

追踪/
  风格决策日志.md        # 每次学习动作追加记录

追踪/schema/
  user-style-profile.json
  user-style-rules.jsonl
  output-pollution-rules.jsonl  # 模型输出污染硬规则：重复填充、占位符污染、术语循环
```

## 规则分层

| 类型 | 例子 | 写入位置 | 使用方式 |
|---|---|---|---|
| 硬约束 | 不接受逐字破折号化；用户确认章节名默认保留；御兽宗不要早登场 | `我的写作偏好.md` + `user-style-rules.jsonl` | 写作前强制带入；可检测项接门禁 |
| 软偏好 | 山脉成长要有试错和受伤；语言要更生活化 | `正文风格画像.md` | 进入 `user_style_constraints` |
| 单书设定 | 本书前期先野外生存，再语言，再交流，再资源认知 | 当前书 `我的写作偏好.md` | 只影响当前项目 |
| 角色语气 | 陈洛心理活动要像野性本能，不像课程学习 | 角色档案或 `正文风格画像.md` | 写对应角色时召回 |
| 禁用表达 | “他终于明白”“这意味着”高频；破折号堆叠 | `禁用表达.md` | 去AI味和正文门禁前检查 |
| 输出污染 | “修真进度阈值”连续重复填充；报告后半段占位符污染 | `output-pollution-rules.jsonl` + `禁用表达.md` + `user-style-rules.jsonl` | 审查/回炉/修复方案落盘后跑 `output-pollution-check.js --learn`，命中即重写 |

## style-learner 输入

调用 `style-learner` 时，主会话必须传入：

```text
项目目录：{dir}
任务描述：学习用户个人风格
用户原话：{最新用户反馈或要求}
证据文件：{可选，用户改过的正文/大纲/审阅反馈/diff 路径}
学习范围：全局作者偏好 / 当前书偏好 / 当前角色口吻 / 当前章节改写偏好
禁止动作：不直接改正文；只更新风格档案、schema 和日志
```

## 输出规则格式

`追踪/schema/user-style-rules.jsonl` 每行一条 JSON：

```json
{"id":"style-20260624-dash-density","scope":"当前书","priority":"hard","category":"标点","rule":"禁止逐字破折号化；少量功能性中文破折号可保留，但不能反复堆砌。","evidence":"用户指出第023章破折号过多。","bad_example":"陈洛的脑子——御——兽——宗——","preferred_fix":"改成动作、短句、身体反应或正常心理流。","source_files":["正文/第1卷/第023章_修士·苏糖.md"],"updatedAt":"2026-06-24T00:00:00+08:00"}
```

字段要求：

- `scope`: `全局作者` / `当前书` / `角色` / `章节`
- `priority`: `hard` / `soft`
- `category`: `结构` / `节奏` / `人物` / `对话` / `标点` / `AI味` / `设定` / `章节名` / `输出污染`
- `rule`: 可执行规则，避免空泛评价
- `evidence`: 用户原话或文件证据摘要
- `bad_example`: 可空；不要粘贴长段
- `preferred_fix`: 用户偏好的替换方向
- `source_files`: 证据路径数组
- `updatedAt`: ISO-like 时间

## 写作前加载

长篇日更、回炉、去AI味、阶段反馈实施前，按顺序加载：

1. `设定/作者风格/我的写作偏好.md`
2. `设定/作者风格/正文风格画像.md`
3. `设定/作者风格/禁用表达.md`
4. `追踪/schema/user-style-rules.jsonl`
5. `追踪/schema/output-pollution-rules.jsonl`
6. 与当前角色/章节相关的 `修改偏好案例.md`

压缩成 `user_style_constraints` 传给 narrative-writer：

```text
user_style_constraints:
- 用户硬约束：...
- 用户软偏好：...
- 本书节奏偏好：...
- 本章/角色相关偏好：...
- 禁用表达：...
- 输出污染硬规则：...
```

## 优先级

1. 平台/合规/安全约束
2. 正文门禁、AI味门禁、章节契约硬约束
3. 用户硬约束
4. 当前书设定和章节细纲
5. 对标文风 / 作者吸收卡
6. 用户软偏好
7. 模型默认写法

用户硬约束不得突破正文门禁；例如用户允许少量 `——`，仍不能允许逐字破折号化、密度失控或 `--`。

## 学习后汇报

每次学习完成后，只汇报：

- 新增/更新了几条规则
- 写入了哪些文件
- 哪些规则会在下次写作自动生效
- 是否有规则只是软偏好，不能作为硬门禁

不要输出内部长日志，不要把用户原文大段复述给用户。
